# 🌙 AnonCare

AnonCare is a premium, anonymous mental health support platform designed to provide safe, professional, and AI-powered care. It combines the power of community support with personalized AI therapy and professional consultations.

## ✨ Key Features

- **Anonymous Community Chat**: Real-time group support using Socket.io with dynamic, alias-based identities.
- **Custom AI Companion**: Personalized AI therapists generated by Google Gemini API based on your specific needs.
- **Professional Consultations**: Browse verified therapists, book appointments, and manage your mental health journey.
- **Private Video Sessions**: Peer-to-peer secure video calling for professional therapy sessions.
- **Premium UI/UX**: High-end glassmorphism design, smooth micro-animations (Framer Motion), and a fully responsive interface.

## 🛠️ Tech Stack

- **Frontend**: React, Vite, Framer Motion, Socket.io-client, Axios.
- **Backend**: Node.js, Express, Socket.io, MongoDB (Mongoose), JWT Authentication.
- **AI Service**: Google Generative AI (Gemini 1.5 Flash).
- **Communication**: WebRTC for peer-to-peer video sessions.

## 🚀 Quick Start

### 1. Prerequisites
- Node.js (v18+)
- MongoDB (Running locally or on Atlas)

### 2. Installation
Clone the repository:
```bash
git clone https://github.com/SauravKumar81/AnonCare.git
cd AnonCare
```

### 3. Environment Setup
Copy the example environment files and fill in your keys:

**Backend (`/server/.env`):**
```env
PORT=5000
MONGO_URI=mongodb://localhost:27017/anoncare
JWT_SECRET=your_jwt_secret
GEMINI_API_KEY=your_gemini_key
FRONTEND_URL=http://localhost:5173
```

**Frontend (`/client/.env`):**
```env
VITE_API_URL=http://localhost:5000
```

### 4. Running Locally
**Start Backend:**
```bash
cd server
npm install
npm run dev
```

**Start Frontend:**
```bash
cd client
npm install
npm run dev
```

## 🌐 Deployment

### Backend (Render)
1. Point to the `server` root directory.
2. Build Command: `npm install && npm run build`
3. Start Command: `npm start`
4. Set environment variables (including `FRONTEND_URL` for CORS).

### Frontend (Vercel)
1. Point to the `client` root directory.
2. Framework Preset: `Vite`.
3. Set `VITE_API_URL` to your Render service URL.
